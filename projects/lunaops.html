LunaOps lets rover logic be tested before hardware enters the room, turning robotics ideas into a controlled simulation workflow.

Why I Built It

To understand robotic navigation through simulation and coordinate-based movement.

Real Problem

Robotic hardware can be expensive or unavailable, but navigation logic still needs validation.

Challenges

Abstracting movement rules and API boundaries without real hardware.

Lessons Learned

Simulation helps test systems early and makes later hardware integration more thoughtful.
